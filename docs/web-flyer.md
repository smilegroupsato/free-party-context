# Web Flyer

The web is a flyer.

It is not the venue.  
It is not the institution.  
It is not the whole participation layer.

The web flyer gives a smell, a hint, an entrance, and the first fragment of a map.

## Domain

```text
eternalfree.party
```

## Published source

The published flyer should be built from:

```text
flyer/
```

The root of the repository should stay clean as context.

## First flyer requirements

The first web flyer should be small.

It should include:

- the name `Eternal Free Party`
- the mountain free party image
- the repository context idea
- a short statement for humans and AI agents
- a link back to GitHub
- a Japanese layer, but not as the only language

## Public link display

Do not display the current GitHub owner name on the public flyer.

Use text such as:

```text
Enter the repository
Repository Contextへ入る
```

The actual URL may point to the current repository for now.
The repository can later be moved to a neutral GitHub Organization.

## Future organization transfer

Planned later:

```text
Current: smilegroupsato/free-party-context
Future: neutral organization / free-party-context
```

The organization name should preserve the world of `Eternal Free Party` and avoid exposing private operator names on the public surface.

## Principle

Do not over-explain.

Put the atmosphere first.
Let the structure be found after entry.
