# Web Flyer

The web is a flyer.

It is not the venue.  
It is not the institution.  
It is not the whole participation layer.

The venue is the repository context.

The web flyer exists to bring someone to the edge.

---

## Core position

```text
web flyer
→ repository context
→ traces
→ repeated behavior
→ expectation
→ naming
→ roles
```

The flyer should not try to contain the whole world.

It should give:

```text
smell
hint
entrance
first map fragment
reason to enter
```

The structure should be found after entry.

---

## Domain

```text
eternalfree.party
```

The domain is the flyer address.

It is not the final address of the venue.

The address is still forming.

---

## Published source

The published flyer should be built from:

```text
free-party-context/flyer/
```

The canonical source for the flyer files is:

```text
rinkaku-no-mae-ni/docs/public/free-party-context/flyer/
```

The root of the repository should stay clean as context.

---

## What the flyer should do

The flyer should:

- name `Eternal Free Party`
- create the feeling of a small mountain site
- say that this is a repository context before the village
- make clear that the repository is the venue
- invite a human or AI agent to enter quietly
- point toward the repository without over-explaining it
- preserve a Japanese layer without making Japanese the only entrance

---

## What the flyer should not do

The flyer should not:

- explain the whole system
- become a landing page for a finished product
- look like a DAO or token project
- look like a startup pitch
- look like a recruitment page
- look like a game world teaser
- claim governance, ownership, rewards, or membership
- expose private operator names as the public surface
- make the website feel like the venue itself

---

## Public link display

Do not display the current GitHub owner name on the public flyer.

Use text such as:

```text
Enter the repository
Repository Contextへ入る
```

The actual URL may point to the current repository for now.

The repository can later be moved to a neutral GitHub Organization.

## Copy principle

Do not over-explain.

Put the atmosphere first.

Let the structure be found after entry.

The flyer should be short enough to leave mystery.

The repository can explain more.

---

## Visual principle

The flyer may suggest:

```text
forest
stream
fog
moss
cold air
cables
temporary lights
an entrance table
a narrow path that is not yet a road
```

It should not become a polished brand site.

It should feel like a flyer found before entering a small mountain gathering.
